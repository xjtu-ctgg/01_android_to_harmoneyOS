目录结构说明
本次大赛分为难题赛道和竞赛赛道，各赛道对应的目录结构如下：

ICT_AI-ARENA/
├── README.md
├── challenges/                 # AI难题
└── competition-problems/       # AI竞赛题
题目说明
在各赛道目录内，按题目序号和名称分别存有各题目的配套工程环境及用例说明。

选手们可根据题目内的README.md了解工程环境和自验证用例的详细情况。

答题要求
本次大赛答题使用线上平台提交作品，在答题截止时间前，可多次上传，以最后一次上传的作品为最终答题作品。

作品形式
作品以zip压缩包形式交付，文件名以 {赛道ID}{题号}{队名}.zip 格式命名，难题赛道ID为01，竞赛赛道ID为02，例如：01_01_攻坚克难队.zip。压缩包解压后应以同名的文件夹为根目录，目录内格式以及关键交付件如下表：

路径	类型	要求	说明
/INSTRUCTION.md	文件	必选	作品运行的入口，平台通过加载该Markdown执行作品运行过程，验证用例通过性
/work	目录	必选	用于存放作品可运行交付件的目录，可以存放Skill/Agent/MCP等
/work/skills/{your-skill-name}/SKILL.md	文件	可选	如提交Skill，按Skill规范存放Skill的目录和入口文件
/work/skills/{your-subagent-name}.md	文件	可选	如提交subagent，在skills目录下存放，并在INSTRUCTION.md中使用
/result	目录	必选	用于存放作品自验证的记录
/result/screenshot	目录	可选	存放作品运行成功、用例通过的截图文件
/result/output.md	文件	必选	记录作品运行成功的输出信息，如有图片，以Markdown规范引用
/logs	目录	必选	用于存放推理过程的记录和交互信息
/logs/interaction.md	文件	必选	记录选手与作品人工交互的记录，如全程无干预，留空
/logs/trace	目录	必选	用于存放推理过程日志
/problem_statement	目录	可选	对于赛道一，难题赛道，如选择自行出题，此目录必须存在，目录内以PROBLEM.md主文件描述自行出题的题目要求
注1：对于赛道一难题赛道，如选择自行出题，题号固定填写00。
注2：对于自行出题的题目，必须包含以下关键章节：
题目背景：题目背景和考核意图的说明
场景价值：题目代表的业务应用痛点、困难，解决的收益
验证工程：作品运行所需的验证环境描述
解题要求：作品的关键得分点，验收标准
用例：作品需要通过的场景和用例


自行出题的题目的PROBLEM.md可参考如下形式
```
## 题目背景
{在这里描述自选题的业务背景}

## 场景价值
{在这里描述自选题所解决的问题的业务价值}

## 验证工程
[{链接名称}](相对PROBLEM.md的验证工程目录，需要一起打包到作品的zip压缩包中)

## 解题要求
1. {评估要求1}
2. {评估要求2}
3. {...}

## 自验证用例
|序号	|输入	|预期输出	|检验方式|
|----|----|----|----|
|01	|{作品需要处理的输入方式}	|{用例期望的标准输出，或输出的关键信息}	|{针对输出准确度的评估方法}|
```
