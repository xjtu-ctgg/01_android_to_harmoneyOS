打分规则说明

一、赛道与赛题规则

目前每个赛道支持选手选择多个赛题进行参赛。

每个团队在同一赛道下可以提交多个赛题作品，但最终每个赛道只会选取一个最优作品作为该赛道的参赛作品。

具体规则如下：

每个赛题的最终得分，以该赛题最后一次成功提交并完成打分的结果为准。
每个赛道的最终参赛作品，为该赛道下所有赛题中得分最高的作品。
若团队同时参与两个赛道并完成打分，评委端最多展示该团队两个赛道对应的最终得分。

二、平台评分方式

打分平台会提供由参赛选手选择的 Agent 框架及模型执行后的客观评分结果，作为评审参考。

当选手选择拉取作品并执行评分时，平台会使用选手选择的 Agent 框架及模型，并行运行 5 个用例集。

三、评分指标说明

1. 用例通过率
用例通过率用于衡量作品在 5 个用例集中的整体通过情况。

用例通过率 = 5 个用例集通过的用例数之和 / 5
​

2. 稳定性
稳定性用于衡量作品在不同用例集下的稳定表现。

稳定性 = 5 个用例集都通过的用例交集
​

3. 准确性
准确性用于衡量作品在单个用例集中的最佳表现。

准确性 = 5 个用例集中通过用例数最高的用例集
​

4. 最终得分
最终得分综合考虑稳定性与准确性。

最终得分 = 100 * （稳定通过的用例数 + 准确通过的用例数）/（用例总数 × 2）
​

四、Agent 框架选择

目前平台支持以下 Agent 框架：


CodeAgent 2.0
OpenCode
