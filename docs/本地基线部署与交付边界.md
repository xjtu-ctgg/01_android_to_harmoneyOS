## 2. 本机构建环境

当前已安装并验证：

- Android SDK：`/Users/ctgg/Library/Android/sdk`
- Android SDK Platform 36
- Android Build Tools 36.0.0
- Android Platform Tools 37.0.0
- Gradle Wrapper 9.4.1（由仓库自带 wrapper 管理）

仓库根目录的 `local.properties` 指向上述 Android SDK。该文件已被上游 `.gitignore` 忽略，不属于交付源码改动。

首次联网构建使用本机代理，并限制为单 worker，避免首次大量 Google Maven 并发下载导致代理 TLS 握手不稳定：

```bash
cd work
env \
  JAVA_HOME=/opt/homebrew/opt/openjdk@21/libexec/openjdk.jdk/Contents/Home \
  ANDROID_HOME=/Users/ctgg/Library/Android/sdk \
  ANDROID_SDK_ROOT=/Users/ctgg/Library/Android/sdk \
  ./gradlew :app:assembleDebug \
  -Dhttp.proxyHost=127.0.0.1 \
  -Dhttp.proxyPort=7897 \
  -Dhttps.proxyHost=127.0.0.1 \
  -Dhttps.proxyPort=7897 \
  --no-parallel \
  --max-workers=1
```

依赖缓存完成后，已使用下列命令完成离线洁净复编译：

```bash
cd work
env \
  JAVA_HOME=/opt/homebrew/opt/openjdk@21/libexec/openjdk.jdk/Contents/Home \
  ANDROID_HOME=/Users/ctgg/Library/Android/sdk \
  ANDROID_SDK_ROOT=/Users/ctgg/Library/Android/sdk \
  ./gradlew clean :app:assembleDebug \
  --offline \
  --no-parallel \
  --max-workers=1
```

验证结果为 `BUILD SUCCESSFUL`。生成的本地验证 APK 位于：

```text

```

APK 包名为 `com.example.jetsnack`，应用名为 `Jetsnack`，min SDK 23，compile/target SDK 36，Debug v1/v2 签名验证通过。该 APK 只用于证明原始 Android 基线可构建，不是最终鸿蒙交付件。
