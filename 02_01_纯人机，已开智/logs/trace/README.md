# 评审证据索引

本目录是提交包中的过程证据，不是平台运行入口。平台运行入口仍是根 `INSTRUCTION.md`，可运行交付件仍是 `work/`。

## 阅读顺序

| 文件 | 重点 | 对应评分关注 |
| --- | --- | --- |
| `01-research-and-baseline.md` | 赛题合同、资料调研、基线事实和方案裁定 | 泛用性、方法合理性 |
| `02-migration-and-architecture.md` | Android 事实到 ArkUI 工程的映射 | 功能、接口、工程完整性 |
| `03-hidden-case-and-quality.md` | 隐藏用例模拟、视觉/交互/规范门禁 | 功能、截图、接口、规范 |
| `04-delivery-and-stability.md` | 平台结果复盘、稳定性专项和归档复核 | 可复现性、性能风险、交付规范 |
| `05-evidence-map.md` | 评委按评分维度快速定位源码证据 | 主观评审效率 |
| `validation.md` | 命令、数量、结果和未验证边界 | 结果真实性 |

## 证据原则

1. 每个结论尽量同时给出源码、机器可读清单、Journey 或测试路径。
2. 本地代理只能证明本地条件下的可重复性，不能代替平台隐藏用例、官方 Code Linter 或真机截图。
3. 日志不包含密钥、个人绝对路径、模型私有思维过程或未经许可的第三方源码。
4. `result/` 与 `logs/` 用于评委参考；最终运行时应以 `work/` 和 `INSTRUCTION.md` 为准。
